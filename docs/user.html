Описание системы · Warehouse API

Описание системы

Warehouse API — серверная часть системы управления складом. Предоставляет REST API для учёта товаров, управления заказами клиентов и поставщиков, контроля остатков, ведения акций и оформления возвратов.

Сервер реализован на языке Go с использованием фреймворка Gin. В качестве базы данных используется PostgreSQL. Все запросы к защищённым ресурсам требуют аутентификации через JWT-токен.

Все запросы и ответы передаются в формате JSON. Базовый URL сервера: http://localhost:8080

Возможности системы

  • Ассортимент — ведение справочника товаров с ценами и артикулами
  • Остатки на складе — учёт количества товаров и дата последнего обновления
  • Заказы клиентов — приём, обновление и отмена заказов с позициями
  • Заказы поставщикам — формирование закупок и учёт поступления товаров
  • Акции — управление скидками и специальными ценами на товары
  • Возвраты — оформление возвратов с привязкой к исходному заказу
  • Аутентификация — регистрация, вход, ролевая система (employee / manager / admin)

Архитектура

Проект разделён на независимые пакеты:

  • config — загрузка конфигурации из .env
  • db — пул соединений с PostgreSQL (pgx/v5)
  • middleware — JWT-аутентификация
  • models — структуры данных всех сущностей
  • handlers — HTTP-обработчики эндпоинтов
Начало работы · Установка и требования

Установка и требования

Требования

  • Go 1.21 или выше
  • PostgreSQL 14 или выше
  • Docker (опционально, для запуска БД)

1. Запуск базы данных

Запустите PostgreSQL через Docker:

docker run -d \ --name warehouse-db \ -e POSTGRES_USER=warehouse \ -e POSTGRES_PASSWORD=secretpassword \ -e POSTGRES_DB=warehouse_db \ -p 5432:5432 \ postgres:16

Примените схему базы данных:

docker exec -i warehouse-db psql -U warehouse -d warehouse_db < schema.sql

2. Настройка

Создайте файл .env в корне проекта:

DB_URL=postgres://warehouse:secretpassword@localhost:5432/warehouse_db JWT_SECRET=your-secret-key-here PORT=8080

3. Запуск сервера

go mod download go run main.go

Сервер запустится на http://localhost:8080

4. Проверка

curl http://localhost:8080/api/health # {"status":"ok"}
При первом запуске в базе создаётся тестовый администратор:
Email: admin@warehouse.local · Пароль: admin123
Начало работы · Аутентификация

Аутентификация

API использует JWT (JSON Web Token). При входе или регистрации сервер возвращает токен, который нужно передавать в заголовке каждого защищённого запроса.

Регистрация

POST /api/auth/register Content-Type: application/json { "email": "user@example.com", "password": "secret123", "first_name": "Иван", "last_name": "Иванов", "phone": "+7 (900) 000-00-00", "role": "procurement_specialist" }

Ответ:

{ "token": "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...", "user": { "id": 5, "email": "user@example.com", "first_name": "Иван", "last_name": "Иванов", "role": "procurement_specialist" } }

Вход

POST /api/auth/login Content-Type: application/json { "email": "user@example.com", "password": "secret123" }

Использование токена

Добавьте токен в заголовок всех последующих запросов:

GET /api/assortment Authorization: Bearer eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...

Прочее · Основные функции

Основные функции

1. Ролевые дашборды

  • Специалист по закупкам — видит истекающие товары, критически низкие остатки и рекомендации к закупке
  • Менеджер по клиентам — видит заказы, ожидающие подтверждения, и товары для акций
  • Кладовщик — видит ожидаемые поставки, заказы к сборке и просроченные товары

2. Управление ассортиментом

  • Ведение справочника товаров с ценами, категориями и артикулами
  • Отслеживание срока годности (expiry_date) и минимального остатка (min_stock_level)
  • Фильтрация: истекающие / низкий остаток / просроченные
  • График продаж за 6 месяцев для каждого товара

3. Складской учёт и прогноз

  • Учёт количества каждого товара с датой последнего обновления
  • Прогноз спроса на основе средних продаж за 3 месяца
  • Автоматические рекомендации по объёму закупки

4. Управление заказами

  • Заказы клиентов: статусы new → confirmed → processing → completed / cancelled
  • Подтверждение и отклонение заказа одним запросом
  • Заказы поставщикам с указанием поставщика, даты доставки и условий
  • Привязка заказов к клиентам и поставщикам из справочников

5. Акции и скидки

  • Статусы акций: active / inactive / expired
  • Автоматические предложения акций по товарам с истекающим сроком
  • Статистика акции: количество заказов и суммарная выручка

6. Поставщики и клиенты

  • Справочник поставщиков с контактной информацией
  • Справочник клиентов с поиском по имени и компании
  • Привязка к заказам для отслеживания истории

7. Возвраты

  • Оформление возврата с указанием причины и списка товаров
  • Привязка возврата к исходному заказу клиента

8. Аутентификация и безопасность

  • Регистрация с выбором роли: procurement_specialist / client_manager / warehouse_keeper
  • Пароли хранятся в виде bcrypt-хеша
  • Доступ к API защищён JWT-токенами
  • CORS настроен для React-фронтенда
